  11. Regarding CENG 317 Hardware Production Technology, a Broadcom development platform will be used by each student who will connect a sensor or effector unit to it. The goal is to scale the hardware to include multiple units and package it to fit in a suitable enclosure that fits within the overall project maximum dimensions of 12 13/16" x 6" x 2 7/8" (32.5cm x 15.25cm x 7.25cm) which represents the space below the tray in the parts kit. A custom I2C PCB with a six pin stackable header will be designed for each sensor/effector. The highest AC voltage that will be used is 16Vrms from a wall adaptor from which +/- 15V or as high as 45 VDC can be obtained. Maximum power consumption will be 20 Watts.
  12. We work with prototypes and prototypes are not to be left powered unattended.
  13. The Raspberry Pi 3 (that you purchase) is a Broadcom development platform that was chosen for a number of reasons. Let us start with platform requirements, it should be able to be used to create a small programmable device capable of connecting to enterprise Wi-Fi, bluetooth, and I2C. The Motorola/Freescale/NXP/Qualcomm HCS12 can have a network stack loaded on it, but then not be able to do much more and thus the device would require multiple HCS12s for this purpose causing an increase in cost. Likewise with Atmel based Arduinos, connecting them to enterprise Wi-Fi is not straightforward. TI's beaglebone shows promise as an alternative however boot times and resources are not as good.

    2. The following case for is used for the parts crib Broadcom development platforms (a Pi 3 also fits) http://www.canakit.com/raspberry-pi-black-case.html and we bought the USB network adaptors from Tiger Direct.
    3. Note that USB webcams are more economical and robust than pi specific camera modules.
    4. More detailed instructions are available on Lynda.com plus Paul Moggach's technical notes. It turns outs that the newest version of Rasbian has realvnc-vnc-server installed by default which conflicts with xrdp.
    5. Instead of installing xrdp on your development platform you can install VNC Viewer on your desktop to connect instead of using Remote Desktop Connection.
    6. The boot options still need to be set along with ensuring VNC (and I2C) is enabled via Menu > Preferences > Raspberry Pi Configuration > Interfaces and such an image is in the DropBoxes as before.
    7. Please let me know if you have had success purging realvnc-vnc-server and installing then using xrdp. Something along the lines of sudo apt-get purge realvnc-vnc-server, sudo apt-get install xrdp, plus relevant configuration.
    8. There is a guide to buying a Raspberry Pi as part of https://www.lynda.com/Raspberry-Pi-tutorials/Up-Running-Raspberry-Pi/191267-2.html. Humber students/staff/faculty/Admin have access to Lynda.com. Access 1000's of online videos ranging from Business, Creative and Technology topics:
